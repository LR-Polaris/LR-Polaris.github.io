## 📘 一、WSL2 介绍与安装

**核心概念：**
WSL（Windows Subsystem for Linux）是 Windows 内置的 Linux 兼容层，WSL2 基于轻量虚拟机运行真实 Linux 内核，支持完整的系统调用兼容性，适合开发环境。

**一键安装（Win10 2004+/Win11，管理员 PowerShell）：**
```powershell
wsl --install          # 默认安装 Ubuntu + WSL2
wsl --install -d Ubuntu-22.04   # 指定版本
wsl --set-default-version 2
```

## 🤖 三、Claude Code（Anthropic Claude CLI）在 WSL 上安装

**前置条件：** WSL2 + Ubuntu，先装 Node.js（推荐用 nvm）

```bash
# 1. 安装 nvm 和 Node.js
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/master/install.sh | bash
source ~/.bashrc
nvm install --lts

# 2. 安装 Claude Code
npm install -g @anthropic-ai/claude-code

# 3. 启动（首次会引导浏览器 OAuth 登录）
claude
```

> WSL2 内浏览器可能无法自动弹出，可设 `export BROWSER='/mnt/c/Program\ Files/Google/Chrome/Application/chrome.exe'` 或直接用 API Key 登录。

## 🔧 四、Synopsys EDA 工具（VCS/DC/Verdi/PrimeTime）在 Linux 上安装

> ⚠️ 注意：Synopsys 工具需持有正版 License 或学校/企业授权包，以下仅为安装环境参考。

**典型前置依赖（Ubuntu）：**
```bash
sudo apt install csh tcsh lsb lsb-core libpng12-0 build-essential flex bison
sudo ln -sf /bin/bash /bin/sh   # EDA工具常要求 sh=bash
```

**大致流程：**
1. 用 Synopsys Installer (setup.sh) 依次安装 SCL → VCS → Verdi → DC → PrimeTime
2. 配置 `~/.bashrc` 添加 `PATH`、`LM_LICENSE_FILE`/`SNPSLMD_LICENSE_FILE`
3. 启动 `lmgrd` 加载 license.dat

> EDA 工具体积巨大（数十 GB），建议在 WSL2 中用 `wsl --export/import` 迁移虚拟磁盘到其他盘。
